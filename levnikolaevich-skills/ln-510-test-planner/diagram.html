Test Task Planner - State Diagram

📋 Overview

  • Purpose: Create final Story task after manual testing passes (invoked by ln-340-story-quality-gate Pass 1)
  • Philosophy: Minimum Viable Testing - Start with 2 baseline E2E tests (positive + negative), add more ONLY with critical justification
  • Realistic Goal: 2-7 tests per Story (hard limit: 28)
  • Output: Comprehensive task with 11 sections: E2E-first Risk-Based Testing (Priority ≥15, tests OUR logic ONLY)
  • Risk Priority: Business Impact × Probability, Priority ≥15 MUST be tested
  • Critical Justification: Each test beyond baseline 2 requires documented answer: "Why does this test OUR business logic (not framework/library/database)?"
  • Delegation: Delegates task creation to ln-311-task-creator (CREATE mode) or ln-312-task-replanner (REPLAN mode) with taskType: "test"
