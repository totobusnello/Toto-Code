## üéØ What's New in v1.2.0

### GOALIE SDK Integration - Goal-Oriented Action Planning

Research-swarm now includes **GOALIE (Goal-Oriented Action Planning)** as a first-class SDK, enabling intelligent goal decomposition and adaptive research planning using GOAP algorithms.

**Key Features**:
- ‚úÖ **Automatic Goal Decomposition** - Complex goals broken into manageable sub-goals
- ‚úÖ **Complexity Estimation** - Sub-goals rated as low/medium/high/very-high
- ‚úÖ **Adaptive Swarm Sizing** - Agent count adjusts based on sub-goal complexity
- ‚úÖ **Multi-Provider Support** - Works with Claude, Gemini, OpenRouter
- ‚úÖ **Graceful Fallback** - Continues with single goal if GOALIE unavailable

   - Set `OPENROUTER_API_KEY`

4. **Custom MCP Servers**
   - Build your own search integrations
   - Specialized data sources
   - Complete flexibility

## üì¶ New CLI Commands

### GOALIE Commands

```bash
# Goal-based research with GOAP planning
research-swarm goal-research "<goal>"
  -d, --depth <number>            # Research depth per sub-goal (default: 5)
  -t, --time <minutes>            # Total time budget (default: 120)
  --swarm-size <number>           # Base swarm size (default: 5)
  --max-concurrent <number>       # Max concurrent agents (default: 3)
  --provider <name>               # AI provider (anthropic|gemini|openrouter)
  --verbose                       # Show detailed output

# Create GOAP plan (planning only, no execution)
research-swarm goal-plan "<goal>"
  --swarm-size <number>           # Base swarm size estimate
  --time <minutes>                # Time budget estimate

# Decompose goal using GOALIE GOAP algorithm
research-swarm goal-decompose "<goal>"
  --max-subgoals <number>         # Max sub-goals (default: 10)
  --verbose                       # Show GOALIE output

# Explain GOAP planning for a goal
research-swarm goal-explain "<goal>"
```

## üìà Performance Impact

### GOALIE Overhead

- **Planning phase**: +30-60 seconds (one-time, per goal)
- **Execution**: Same as v1.1.0 (parallel swarm execution)
- **Overall**: Minimal impact, intelligent decomposition worth the cost

### Adaptive Sizing Benefits

Complexity-based swarm sizing optimizes cost/quality:

| Sub-Goal Complexity | Agents | Cost vs Fixed 5-agent |
|-------------------|--------|----------------------|
| Low | 3 | 40% savings |
| Medium | 5 | Same |
| High | 6 | +20% |
| Very High | 7 | +40% |

**Result**: Better resource allocation, ~15-30% cost reduction on average

```bash

# Automatic grounding enabled!
```

```bash
# Get Brave API key: https://brave.com/search/api/
export BRAVE_API_KEY="your-key"

# Create mcp-config.json
echo '{
  "mcpServers": {
    "brave-search": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-brave-search"],
      "env": {
        "BRAVE_API_KEY": "'$BRAVE_API_KEY'"
      }
    }
  }
}' > mcp-config.json

export MCP_CONFIG_PATH="./mcp-config.json"
```

## üéì Use Cases

### When to Use GOALIE (`goal-research`)

‚úÖ **Use GOALIE when**:
- Research goal is complex and multi-faceted
- You need intelligent task decomposition
- Sub-goals have varying complexity
- You want adaptive resource allocation
- Planning is as important as execution

**Examples**:
- "Comprehensive analysis of quantum computing applications"
- "Multi-year technology trend forecasting"
- "Cross-domain research (e.g., AI + healthcare + ethics)"

### When to Use Standard Swarm (`research`)

‚úÖ **Use standard swarm when**:
- Goal is already well-defined
- Single focused research task
- You know the optimal agent count
- Fast execution is priority

**Examples**:
- "What are REST APIs?"
- "Compare microservices vs monolithic architecture"
- "Analyze specific technology X"

### When to Use Single-Agent (`--single-agent`)

‚úÖ **Use single-agent when**:
- Very simple query
- Budget is highly constrained
- Single perspective sufficient

**Examples**:
- "Define term X"
- "List features of Y"
- "Quick fact-checking"
