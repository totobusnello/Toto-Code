Story Reviewer - State Diagram

📋 Overview

  • Purpose: Two-pass review - Pass 1 (Early Exit Pattern) + Pass 2 (verify tests + Story Done)
  • Pass 1 (6 phases): Discovery → Preparation → **Code Quality (FAIL FAST)** → **Regression Check (FAIL FAST)** → **Manual Testing (FAIL FAST)** → Verdict
  • Pass 2 (3 phases): Prerequisites check → Test verification (10-28 tests, Priority ≥15) → Verdict and Story closure
  • Early Exit Pattern: Each Pass 1 phase can stop execution and create fix/refactor task
